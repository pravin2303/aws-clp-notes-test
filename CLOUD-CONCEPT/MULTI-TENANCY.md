## Multi-Tenancy – CLF-C02 Scope 🏢☁️

**Multi-tenancy** means multiple customers (tenants) share the same cloud infrastructure, while their data remains isolated and secure.

This is a core concept of cloud computing in  
Amazon Web Services.

---

## Simple Meaning

- Many customers use the same physical servers
    
- Each customer’s data is isolated
    
- Resources are logically separated
    

---

## Why It Is Important

- Reduces cost (shared infrastructure)
    
- Improves resource utilization
    
- Enables economies of scale
    

This is why cloud is cheaper than owning private data centers.

---

## Example

In AWS:

- Multiple companies run EC2 instances
    
- They share physical hardware
    
- But cannot access each other’s data
    

---

## Multi-Tenancy vs Dedicated Host (Exam Concept)

|Feature|Multi-Tenancy|Dedicated Host|
|---|---|---|
|Infrastructure|Shared|Single customer only|
|Cost|Lower|Higher|
|Use Case|General workloads|Compliance requirements|

---

## Exam Keywords 🎯

- “Shared infrastructure” → Multi-tenancy
    
- “Logical isolation” → Multi-tenancy
    
- “Economies of scale” → Multi-tenancy
    

---

### For CLF-C02 remember:

> Multi-tenancy = Shared infrastructure with secure isolation
